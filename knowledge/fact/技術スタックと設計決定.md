# プロジェクト技術スタックと設計決定

## 技術スタック
- バックエンド: Python 3.13 / FastAPI / asyncio
- LLM: Anthropic API (Opus 4.6, Sonnet 4.6) + Google AI Studio (Gemini 2.5 Pro)
- フロントエンド: HTML5 + Vanilla JS + CSS (ダークテーマ)
- 通信: WebSocket（リアルタイム思考ストリーミング）
- データ永続化: ファイルシステム (JSON)
- データバリデーション: Pydantic v2

## 主要設計決定
1. Claude Agent SDKではなく直接Anthropic API呼び出しを採用（Prompt Caching手動制御のため）
2. Phase A-2のWorkerはMVP段階で統合実行（14ファイル分割ではなく1ファイル内で4並列タスク）
3. 日次ループのRIM（Reflective-Impulsive Model）は asyncio.gather で並列実行
4. 隠蔽原則はコンテキストフィルタリング関数で実装（プロンプト構築時にフィルタ）
5. 1日あたりのイベント数は2-4件（合計14-28件）、翌日予定は必ずイベント化（フォールバック付き）
6. 日記は約400字（500字以下）統一。Day1のみ世界観・設定紹介セクションを含む特別指示あり
7. LinguisticExpression（言語的表現方法）はDay 0で生成されるが、日記生成プロンプトにのみ注入。Phase A-2/A-3/Dには渡さない。VoiceFingerprint（具体的特徴）+ 抽象的雰囲気 + 日記書き方の空気感の3層構造
8. エージェンティック生成: Creative Director/Phase A-3/Phase D Step 5は全てエージェンティックループ（draft→critique→self_reflect→submit）で品質担保。2層自己批判（外部批評+内省）により「本当にこれでいいのか」を確認するまでsubmit不可
9. Creative Directorはmin_research_searches（プロファイル設定可能: high_quality=5, standard=3, fast=2, draft=1）回以上のWeb検索を必須化。5フェーズ構造: 計画→リサーチ→ドラフト→批評→自己内省→提出
