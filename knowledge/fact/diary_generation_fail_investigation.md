# 日記生成失敗に関する調査結果

## 概要
日記の生成が失敗し、フォールバックテキスト（「本日は何も書く気になれなかった」等）が表示される問題についての調査結果。

## 事実関係
調査の結果、以下の不具合が特定された。

1. **エージェンティックループの終了判定ロジックの不備**:
   - `backend/tools/llm_api.py` の `call_llm_agentic` 系関数において、AIが `submit_` で始まるツールを呼び出した直後に、ループを強制終了するロジックになっている。
   - 日記生成においては、`submit_final_diary` を呼ぶ前に `check_diary_rules` などのバリデーションを通過している必要があるが、これらをスキップして `submit_` を呼んだ場合に、ツール側が「FAILED」を返しても、システム側は「終了」と判断してループを抜けてしまう。
   - この結果、最終的な日記の内容が保存されないまま処理が終了し、フォールバック処理が働いてしまう。

2. **Gemini API 応答パーツの処理不備**:
   - `call_llm_agentic_gemini` において、`parts[0]` のみを確認して関数呼び出し（function_call）を判定している。
   - AIが思考テキストと関数呼び出しを同時に返した場合、最初がテキストパーツだと関数呼び出しが無視される。

3. **Gemini の複数関数呼び出しへの未対応**:
   - 一度のターンで複数の関数を呼び出す（Parallel function calling）Geminiの挙動に対応できていない。

## 結論
`llm_api.py` のエージェンティックループにおける終了条件の厳格化（ツールが SUCCESS を返した場合のみ終了する）と、Gemini のマルチパート応答への対応が必要である。
